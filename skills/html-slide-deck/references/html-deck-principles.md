# HTML 簡報設計原則

## 頁面角色

交替使用問題頁、比較頁、流程頁、案例頁、圖表頁、迷思頁、總結頁，避免每頁都是相同卡片。

## 視覺系統

- 使用一個背景色、一個文字主色與一至兩個強調色。
- 建立明確的標題、內文、註解與頁碼層級。
- 以留白、對齊與大小建立閱讀順序，不堆疊裝飾陰影。

## 教學互動

- 每三到五頁安排提問、判斷或短任務。
- 互動必須有清楚回饋，不因動畫影響理解。
- 講者備註補充口頭說明，不重複畫面文字。

## 技術檢查

- 所有路徑使用相對路徑。
- 避免將內容直接插入 `innerHTML`。
- 列印時每張投影片獨立分頁。
- 行動裝置可閱讀，但主交付仍以投影畫面為準。
